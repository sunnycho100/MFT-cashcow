## Phase 2: Self-Healing ML Loop

### Step 2.1 — Drift Detection Module (`v2/src/models/drift_detector.py`) **[NEW FILE]**

4-type drift monitor using `river` (online) and `scipy` (batch):
- **Feature drift (ADWIN)**: one detector per monitored feature. Flags when ≥3 features drift simultaneously. Severity: "severe" at 2x threshold
- **Performance drift (Page-Hinkley)**: feeds trade P&L, detects mean shift downward. Always "severe"
- **Calibration drift**: rolling Brier score `(pred - actual)²` over 50-bar window. Flags when increase > 0.05 above baseline. "mild" severity
- **Regime shift (KS-test)**: `scipy.stats.ks_2samp` comparing recent ATR/ADX distributions against training baseline. "severe" if both shift, "mild" if one

API: `set_monitored_features()`, `set_training_baseline()`, `update_features()`, `update_performance()`, `update_calibration()`, `check_regime_shift()`, `get_pending_signals()`, `has_any_drift()`, `reset()`, `get_status()`

### Step 2.2/2.3 — Retrain Manager (`v2/src/models/retrain_manager.py`) **[NEW FILE]**

Orchestrates the full retrain lifecycle:

**7 trigger conditions** (checked in priority order):
1. Feature drift (from DriftDetector)
2. Performance drift
3. Calibration drift
4. 5+ consecutive losing trades
5. Rolling 30-trade Sharpe < -0.5
6. No trades for 14+ days
7. Scheduled weekly (Sunday 00:00 UTC)

**Champion/Challenger evaluation** — ALL must pass:
- `profit_factor`: challenger > champion
- `sharpe`: challenger >= champion
- `max_drawdown`: challenger <= 1.2x champion
- `brier_score`: challenger <= champion

**Safeguards:**
- Cooldown: 48h between retrains (extended to 72h on rejection)
- Emergency halt: 3 consecutive failed/rejected retrains → halt everything
- Missing metrics gracefully skipped in evaluation

### Step 2.4 — LightGBM Model Extensions (`v2/src/models/lgbm_model.py`)

New methods added (existing methods untouched):
- `save_champion()` — timestamped `champion_YYYYMMDD.pkl` + `_meta.json`
- `retrain(train_df, test_df, feature_names)` — saves champion first, then full retrain
- `refine(train_df, test_df, feature_names, n_trees=100)` — incremental via `init_model`. Guards against exceeding 2000 total trees (falls back to full retrain)
- `get_model_version()` — dict with trained_at, n_trees, n_features, metrics
- `list_champions()` — glob for champion checkpoints, sorted newest-first
- `load_champion(path)` — restore from specific champion checkpoint
